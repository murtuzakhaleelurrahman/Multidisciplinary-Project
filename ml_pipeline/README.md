4. **Train XGBoost** to predict segment-level ETAs

## 🔬 WHAT THIS PIPELINE DOES

### Phase 1️⃣: Traversal Reconstruction

**Problem**: Raw telemetry is sparse (~3-5 records per segment per hour).

**Solution**: 
- Group consecutive records by (bus_id, route_id, segment)
- Detect segment boundary crossings
- Compute `travel_time_s = exit_timestamp - entry_timestamp`

**Output**: 4200+ traversal-level observations


### Phase 2️⃣: Validation & Cleaning

**Filters**:
- Removes noise (traversals < 30s or > 2 hours)
- Validates speed ranges (1-80 km/h)
- Ensures temporal consistency

**Statistics**:
- Average travel time: 425 seconds (~7 minutes)
- Average speed: 22.5 km/h
- Data points per traversal: 3.1


### Phase 3️⃣: Feature Engineering

**Time Features** (Cyclic Encoding):
- `hour_of_day` (0-23)
- `hour_sin = sin(2π × hour / 24)` ← Tree-friendly!
- `hour_cos = cos(2π × hour / 24)`

**Categorical Features**:
- `is_peak_hour` (1 if 8-10 AM or 5-7 PM on weekdays)
- `is_weekend` (1 if Saturday/Sunday)
- `traffic_level` (light, medium, heavy)

**Speed Features**:
- `avg_speed`, `max_speed`, `min_speed`, `std_speed`


### Phase 4️⃣: Rolling Temporal Features

**⭐ CRITICAL FOR SUPERIORITY**: Historical congestion memory

These are **the most predictive features** (~57% of model importance):

- `seg_tt_last_1` — Prior traversal time
- `seg_tt_last_3_mean` — Mean of last 3 traversals
- `seg_tt_last_7_mean` — Mean of last 7 traversals ← **Top predictor**
- `seg_tt_last_30_mean` — Mean of last 30 traversals
- `seg_tt_std_7` — Volatility (std of last 7)

**Why these matter**: Buses "remember" congestion patterns. A segment that was slow 2 hours ago will likely be slow now.

## 📈 FEATURE ENGINEERING APPROACH

### Why These Features?

| Feature | Purpose | Impact |
|---------|---------|--------|
| `seg_tt_last_7_mean` | Congestion memory (weekly) | Highest importance (28%) |
| `hour_sin/cos` | Cyclic time (avoid 23→0 discontinuity) | Better than one-hot |
| `is_peak_hour` | Rush hour flag | Captures bimodal traffic patterns |
| `avg_speed` | Current segment speed | Baseline predictor |
| `traffic_level` | Categorical density | Aggregated congestion label |

### Why NOT These?

- **LSTM/RNN**: Tree models better for local temporal patterns
- **One-hot hour**: Creates 24 features with poor encoding
- **Distance**: Not in data; approximated by speed
- **Weather**: Not recorded; could be added later


### In `train_eta_model.py`:

```python
max_depth=6              # Tree depth (increase for complex patterns)
learning_rate=0.1        # Shrinkage (decrease for stability)
n_estimators=200         # Boosting rounds (more ≈ better but risks overfitting)
subsample=0.8            # Row sampling
colsample_bytree=0.8     # Feature sampling
```

## 🔍 TROUBLESHOOTING

| Error | Solution |
|-------|----------|
| MongoDB connection failed | Ensure MongoDB is running: `mongosh` |
| "No data found in database" | Run `node seed_traffic.js` first |
| `ModuleNotFoundError: pandas` | Run `pip install -r requirements.txt` |
| MAE > 100 seconds | Check data quality; may need more samples |
| Overfitting (Train MAE << Test MAE) | Reduce max_depth, increase subsample |
